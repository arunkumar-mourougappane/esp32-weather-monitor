# Release Notes — v3.0.0

This is a major release combining a large wave of quality-of-life UI improvements, multi-network WiFi roaming, AES-256-CTR NVS credential encryption, and a comprehensive set of security and robustness fixes identified in a thorough code review.

---

## 🚀 Highlights

* **AES-256-CTR NVS encryption** — all sensitive credentials are hardware-bound and encrypted at rest
* **Multi-network WiFi roaming** — up to 5 SSIDs ranked by RSSI; fast-connect cache keyed per-SSID
* **Security hardening** — re-provisioning PIN verification, 3-strike lockout, PIN brute-force reboot
* **Hourly Forecast page** — full 24-hour strip with icons, temps, precipitation, and wind
* **Swipe-up detail overlay** — dew point, AQI with EPA label, and active weather alert
* **Moon phase widget & wind rose compass** on the Dashboard
* **Configurable sync interval & battery-adaptive rate**
* **13 bug fixes** across weather fetching, WiFi caching, e-ink rendering, and watchdog handling

---

## ✨ New Features

### AES-256-CTR NVS Credential Encryption
All sensitive values written to non-volatile storage (`wifi_pass_N`, `api_key`, `webhook_url`) are encrypted with AES-256-CTR before being persisted. The 32-byte key is derived from the device-unique eFuse MAC address via SHA-256, making the ciphertext hardware-bound — an NVS dump is unreadable on any other chip. A random 16-byte IV from the ESP32 hardware TRNG is generated on every write; the stored format is `"E1:" + Base64(IV ∥ ciphertext)`. Existing plaintext NVS values are migrated transparently on first boot with no re-provisioning required.

### Multi-Network WiFi with RSSI-Based Selection
Up to 5 SSID/password pairs can be stored in NVS. `WiFiManager::connectBestSTA()` performs an active scan, ranks all matching SSIDs by RSSI, and connects to the strongest available network. The fast-connect cache (BSSID + channel in `RTC_DATA_ATTR`) is keyed to the last connected SSID and reused only when that SSID is still in the candidate list.

### Provisioning Portal — Multi-WiFi UI
The WiFi fieldset is replaced with a dynamic list. Up to 5 networks can be added with **+ Add Network** or removed with **× Remove**. Index 0 is labelled as primary. The `ProvisionSaveCallback` type is simplified to `std::function<void(const WeatherConfig&)>`.

### Hourly Forecast Page
A new 4th page (accessible by swiping) shows a complete 24-hour strip from the Open-Meteo hourly endpoint: time in 12-hour format, vector weather condition icon, temperature, precipitation chance, and wind speed per card. Grid cell borders separate each hour column.

### Swipe-Up Detail Overlay
A vertical swipe reveals a full-screen overlay with information not visible on the main Dashboard:
- AQI value with EPA descriptive category label (Good / Moderate / Unhealthy…)
- Active weather alert headline + severity, or a reassurance string when none is active
- Estimated dew point calculated from temperature and humidity

### Moon Phase Widget
A fractional moon phase glyph derived from Unix time is rendered on the Dashboard, updated on every sync cycle.

### Wind Rose Compass
The Dashboard replaces the raw text bearing with an 8-point Cartesian compass dial showing the prevailing wind direction as a vector indicator with speed label.

### Configurable Sync Interval
A sync interval selector is exposed in the provisioning portal. The deep-sleep timer respects the user-configured value instead of the hardcoded 30-minute default.

### Battery-Adaptive Sync Rate
If the battery voltage drops below 40% (≈3650 mV), the sync interval is automatically doubled to extend battery life without user intervention.

### Double-Click Webhook
A physical double-tap of the G38 button fires an HTTP GET to a user-configured webhook URL and shows a brief confirmation message on screen. Configurable in the provisioning portal.

## 🐛 Bug Fixes

### WeatherService
- **Supplemental data blocked by current-conditions failure** — a failed primary fetch previously caused an early `return`, skipping all supplemental fetches (AQI, sun times, hourly, alerts). A `currentOk` flag now allows supplemental fetches to proceed regardless.
- **No HTTP timeout** — all five sequential fetches could block indefinitely on a dead server. Added `http.setTimeout(10000)` to prevent the device staying awake permanently.
- **`DailyForecast::dayTime` never populated** — the timestamp was declared but never set, causing the Forecast page to show zeroed weekday labels. Now parsed from `interval.startTime` (RFC 3339) via `strptime` / `mktime`.

### WiFiManager
- **Stale RTC fast-connect cache on timeout** — `connectSTA` and `connectBestSTA` now zero `rtc_bssid` and clear `rtc_cached_ssid[0]` on timeout. Previously only `rtc_channel` was cleared, leaving a stale BSSID that caused the next boot to attempt a dead fast-connect.

### WebServer / Security
- **Re-provisioning without PIN verification** — any client on the open SoftAP could POST `/save` and overwrite config. The handler now loads `pin_hash` from NVS and, if set, requires a matching `current_pin` field. Three wrong submissions trigger a 60-second lockout (HTTP 429).
- **Sparse WiFi slot halts provisioning** — a missing param index was using `break`, silently dropping all higher-indexed networks. Changed to `continue`.

### ConfigManager
- **Silent NVS write failures** — `putInt` / `putString` return values are now checked for critical keys; `ESP_LOGE` is emitted when a write returns zero bytes, making flash exhaustion or NVS corruption visible in the log.

### AppController
- **Webhook GET no timeout** — added `http.setTimeout(5000)` before the webhook `http.begin()` call.

### DisplayManager
- **Duplicate `_drawBattery()` call** — `drawPageDashboard()` called `_drawBattery()` even though `renderActivePage()` already calls it before dispatch. Removed the redundant call.
- **Task watchdog reset missing in PIN loop** — `promptPIN()` now calls `esp_task_wdt_reset()` on every iteration and returns an empty string after a 2-minute idle timeout, preventing both a WDT reset and an indefinitely locked display.

### main
- **Unlimited PIN retry attempts** — the provisioning PIN gate had no brute-force limit. The device now reboots after 3 consecutive wrong PIN entries.

### UX Polish
- Dashboard detail labels expanded: `Humidity`, `UV Index`, `Visibility`.
- `showMessage()` font corrected to `FreeSansBold18pt7b` / `FreeSans12pt7b`.
- Forecast precipitation label renamed `Rain` → `Precip`.
- Forecast page `"Swipe for more"` hint repositioned right-aligned to avoid conflicting with the back-arrow.
- Settings diagnostics reformatted as a two-column label/value layout with a `Last synced` row.
- Pagination dots show active page name label above the strip.
- Page switches show a brief horizontal stripe flash for visual swipe feedback.
- `updateClockOnly` font-number argument removed — clock now renders in the correct large bold font on every minute tick.

## 🔧 Code Quality

- **EXT0 wakeup helper extracted** — the four-line `rtc_gpio_init` / `rtc_gpio_set_direction` / `rtc_gpio_pulldown_dis` / `esp_sleep_enable_ext0_wakeup` block was duplicated in three sleep-entry paths. Extracted into `_configureEXT0Wakeup()`.
- **`kForecastColsPerView` constant** — magic literal `3` replaced by a named constant.
- **Ghost cleanup interval raised 20 → 48** — reduces disruptive full-panel flash during normal usage (~24-hour cadence at 30-minute sync intervals).

## ⬆️ Upgrade Notes

1. On first boot after flashing v3.0.0, `ConfigManager::begin()` automatically re-encrypts any legacy plaintext NVS credentials. No re-provisioning is required unless you also wish to add additional WiFi networks.
2. If a provisioning PIN was set in v2.x, the new `/save` handler will require that PIN to be re-submitted as `current_pin` when reconfiguring via the portal.
3. The `APP_VERSION` build flag in `platformio.ini` is now `3.0.0`.

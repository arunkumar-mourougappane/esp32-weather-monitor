# Release Notes — v2.0.0

This is a major feature release adding a significantly richer UI, touch-driven navigation, an animated loading screen, two new forecast charts, and a set of hardware bug fixes that were silently preventing correct device behaviour.

---

## 🚀 Highlights

* **Fully reworked display UI** across all three pages — Today, 10-Day Forecast, and Settings & Diagnostics
* **Touch-driven settings menu** with vector icon grid
* **Animated 3-step loading screen** with cloud+sun illustration
* **Critical hardware fix** — G38 button wakeup from deep sleep now works correctly
* **Accurate diagnostics** — battery voltage and last-known IP now always show real data

---

## ✨ New Features

### Animated Loading Screen
When the device has no cached data and performs its first sync, a full `epd_quality` splash is shown:
- City name header
- Cloud + sun vector illustration (rendered using a fill→hollow technique to produce a clean outline with no arc seams)
- 3-step progress indicator — **Connecting to WiFi → Syncing Time → Fetching Weather** — with a fill progress bar, step dots, checkmarks for completed steps, and a current action label
- Each step advance uses a fast `epd_fastest` partial refresh so the illustration never redraws

### Icon-Based Settings Menu
The Settings & Diagnostics page now shows a **3-column vector icon grid** instead of a text list:
- **Sync** — circular-arrow icon → triggers an immediate Force Sync
- **Setup** — concentric-arc WiFi icon → launches the provisioning portal
- **Sleep** — crescent-moon icon → puts the device to sleep immediately
- Tap the column to trigger; the active item highlights with an inverted (white-on-black) treatment

### Touch Tap Detection
`InputManager::checkTap(x, y)` detects a touch release that did not cross the 30 px swipe threshold, enabling tap-based menu interaction without conflicting with horizontal swipe scrolling for the forecast page.

### Today Page Enhancements
- **Wind bearing**: 8-point compass direction appended to wind speed (e.g. `Wind: 14 km/h NE`)
- **UV index and visibility**: new third row in the details grid
- **Sunrise and sunset times**: displayed flanking the Sun Arc dial (e.g. `06:42 ··· Sun ··· 18:15`)
- **Tomorrow preview card**: fully redesigned — centred, with weather icon, condition text, H/L temperatures, and precipitation chance

### 10-Day Forecast Page Enhancements
- **Temperature band sparkline**: dual-line chart plotting daily highs (thick) and lows (thin) across all 10 days, with Y-axis, Hi/Lo legend, and degree symbol annotations
- **Precipitation bar chart**: new `_drawPrecipBars()` vertical bar chart showing rain probability (0–100 %) per day, on the same horizontal day grid as the temperature chart
- **Enriched forecast cards** (3 per view, swipe-scrollable):
  - Real weekday name from the forecast `dayTime` timestamp (e.g. `Mon 09`)  
  - Vector weather icon per card
  - H/L temperature text
  - Temperature range bar contextualised against the full 10-day span
  - Precipitation chance always shown
  - Vertical column dividers between cards

### Last-Known IP Persistence
The device now caches the last successfully-assigned WiFi IP address in `RTC_DATA_ATTR` memory, so it survives deep sleep. The Settings diagnostics row shows:
- `IP: 192.168.1.42` — WiFi is currently live
- `IP: 192.168.1.42 (offline)` — cached from the last sync cycle (normal state during button-wakeup interactive mode)
- `IP: No data yet` — device has never connected

### Force Sync Immediate Wakeup
The **Force Sync** action now calls `_enterDeepSleepForImmediateWakeup()`, which uses a 1-second sleep timer instead of the normal 30-minute interval. The device wakes immediately into the standard timer-fetch path, so weather data refreshes within seconds.

## 🐛 Bug Fixes

### G38 Button Wakeup Never Fired
**Root cause:** `rtc_gpio_init()` was never called before `esp_sleep_enable_ext0_wakeup()`. Without this call the pin remains in the Arduino GPIO-matrix domain and the RTC peripheral cannot see signal changes during deep sleep — causing all button presses to be silently ignored.

Additionally, `rtc_gpio_pullup_en()` was called on `GPIO_NUM_38`, which silently fails for GPIO34–39 (input-only pins with no internal pull resistors on the ESP32).

**Fix:** Added `rtc_gpio_init()` + `rtc_gpio_set_direction(RTC_GPIO_MODE_INPUT_ONLY)` before the EXT0 registration. Removed the failing `rtc_gpio_pullup_en()` call (the M5Paper PCB has an external pull-up on G38). Applied to both `enterDeepSleep()` and `_enterDeepSleepForImmediateWakeup()`.

### Screen Stuck on "Fetching Weather"
When woken by the G38 button with no cached data, the display was rendering the "Fetching weather…" placeholder — implying a fetch was in progress when it was not. Fixed by displaying a `showMessage("No Data Yet", "Weather syncs every 30 min")` screen and adding a `rtcCachedWeather.valid` guard around `renderActivePage` in the interactive session loop.

### Force Sync Stuck on "Syncing…"
Force Sync was calling `enterDeepSleep()` (30-minute timer), causing the device to sleep for 30 minutes with "Syncing…" on screen. Fixed by using `_enterDeepSleepForImmediateWakeup()` (1-second timer).

### Battery Shows 0.00 V
The Settings diagnostics page was calling `M5.Power.getBatteryVoltage()`, which always returns 0 on this hardware revision. Fixed by using `_cachedBatVoltage` and `_cachedBatLevel`, which are populated by the working `analogReadMilliVolts(35)` path (via a 1/2 voltage divider on GPIO 35).

### Settings IP Always Showing "Offline"
WiFi is not connected during interactive (button-wakeup) sessions, so `WiFi.localIP()` always returned `0.0.0.0`. Fixed by caching the IP into `RTC_DATA_ATTR char rtcLastIP[16]` immediately after each successful WiFi connect; this value persists through deep sleep cycles.

## 🔄 Changed Behaviour

| Area | Old behaviour | New behaviour |
|---|---|---|
| Settings navigation | G38 rocker click | Capacitive tap on icon column |
